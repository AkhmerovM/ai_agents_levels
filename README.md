# Учебный стенд «Уровни ИИ-агентов»

Многомодульное Spring Boot приложение, которое наглядно демонстрирует три уровня работы с LLM.
Пишешь запрос в чат, выбираешь паттерн — и видишь, что происходит «под капотом».

## Стек

- Java 21 + Spring Boot 3.3 + Maven (multi-module)
- LangChain4j 0.35 — абстракции для работы с LLM
- OpenRouter — прокси к LLM (совместим с OpenAI API)
- all-MiniLM-L6-v2 — локальные эмбеддинги (в JVM, без API)
- PostgreSQL 16 + pgvector — векторное хранилище для RAG
- Vanilla JS фронтенд (без фреймворков)

## Быстрый старт (Docker)

### 1. Получить ключ OpenRouter

1. Зайди на [openrouter.ai](https://openrouter.ai)
2. Зарегистрируйся → Keys → Create Key
3. Скопируй ключ

### 2. Настроить окружение

```bash
cp .env.example .env
# Открой .env и вставь ключ в OPENROUTER_API_KEY=
```

### 3. Запустить

```bash
docker compose up --build
```

Первый запуск займёт 3-5 минут (загрузка зависимостей Maven).

### 4. Открыть

```
http://localhost:8080
```

## Локальный запуск (без Docker)

Нужен PostgreSQL с расширением pgvector и Java 21.

```bash
# Переменные окружения
export OPENROUTER_API_KEY=sk-or-...
export LLM_MODEL=openai/gpt-4o-mini
export DB_HOST=localhost
export DB_USER=ailevels
export DB_PASSWORD=ailevels

# Запуск
cd app
mvn spring-boot:run
```

## Паттерны по уровням

### Уровень 1 — Чистая LLM

Показывает, как формулировка промпта влияет на результат. Память не используется.

| Паттерн | Что демонстрирует | Попробуй спросить |
|---------|-------------------|-------------------|
| **Zero-Shot** (базовая линия) | Вопрос без приёмов — с чего начинается сравнение | «Что такое машинное обучение?» |
| **Few-Shot** | Примеры в промпте учат модель без дообучения | «Доставили быстро, товар отличный!» (тональность) |
| **Chain-of-Thought** | Инструкция «думать вслух» улучшает логические задачи | «У Ани 5 яблок, она отдала половину...» |
| **Role Prompting** | System-промпт задаёт роль и стиль общения | «Объясни квантовую запутанность» (смени роль!) |

### Уровень 2 — Память и RAG

Подключается PostgreSQL + pgvector. Для RAG сначала нажми **«Загрузить документы»**.

| Паттерн | Что демонстрирует | Попробуй спросить |
|---------|-------------------|-------------------|
| **RAG** | Ответы на основе твоих документов, а не обучающих данных | «Как вернуть товар?», «Когда работает магазин?» |
| **Sliding Window** | Последние N сообщений хранятся в памяти | Представься, задай 5 вопросов, спроси «как меня зовут?» |
| **Summarization** | Старая история сворачивается в резюме | Расскажи о себе 5-6 фактов по одному — увидишь суммаризацию |

### Уровень 3 — Инструменты ⭐

Главный уровень. Модель сама вызывает функции.

| Паттерн | Что демонстрирует | Попробуй спросить |
|---------|-------------------|-------------------|
| **Function Calling** | Полный пинг-понг: запрос→tool_call→результат→ответ | «Погода в Сочи и сколько будет 123*456?» |
| **Structured Output** | LLM возвращает JSON-объект вместо текста | «Ужасный товар, сломался через день, не куплю снова» |
| **Schema-based Routing** | Модель сама выбирает инструмент — без if/else в коде | Поочерёдно: погода / математика / время — смотри выбор |

## Колонка «Под капотом»

Это ключевая обучающая часть. Каждый запрос возвращает `trace` — список шагов:

| Иконка | Тип | Что показывает |
|--------|-----|----------------|
| 📝 | prompt | Итоговый промпт, отправленный в LLM |
| 🤖 | model | Ответ от LLM |
| 🔧 | tool-call | Запрос модели на вызов инструмента |
| ✅ | tool-result | Результат выполнения инструмента |
| 🔍 | retrieved | Чанки из векторного хранилища (RAG) |
| 🧠 | memory | Сообщения в окне памяти |
| ℹ️ | info | Пояснения и метаданные |

Кликни на заголовок шага, чтобы свернуть/развернуть.

## Примечания к реализации

- **OpenRouter вместо OpenAI напрямую**: подключается через `langchain4j-open-ai` с `baseUrl=https://openrouter.ai/api/v1`. Одного ключа достаточно для всех моделей.
- **Эмбеддинги локально**: OpenRouter не поддерживает `/embeddings`, поэтому RAG использует `AllMiniLmL6V2EmbeddingModel` — работает в JVM без API.
- **Модель для function calling**: убедись, что выбранная `LLM_MODEL` поддерживает tool use. `openai/gpt-4o-mini` и `anthropic/claude-haiku` поддерживают.
